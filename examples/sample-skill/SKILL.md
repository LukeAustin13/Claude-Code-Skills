# Sample Skill — Summarize File

This is an example of a well-formed SKILL.md. Copy this as a starting point for new skills.

---

## When to use
When asked to summarize a file, document, or piece of code for someone unfamiliar with it.

## What to do
- Read the target file in full before writing anything.
- Identify the purpose, key components, and anything that would surprise a new reader.
- Write a short summary that a developer with no prior context could act on.

## Constraints
- Do not copy-paste large blocks of the file into the summary.
- Do not make inferences beyond what is in the file.
- Keep the summary under 200 words unless the file is very large.

## Output style
Plain prose, 2-4 short paragraphs. No bullet lists unless listing distinct components. No headers unless the file has clearly separate sections.
